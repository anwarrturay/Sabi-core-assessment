# F1 — Study Screener

A React (JSX) + Vite interface for title/abstract screening. A reviewer scrolls
through research studies, reads each abstract, and marks it **Include**,
**Exclude**, or **Flag for discussion**. Decisions persist across refreshes, and
the whole workflow is operable without a mouse.

## Run it

```bash
cd F1
npm install
npm run dev        # opens http://localhost:5173
```

Build / preview a production bundle:

```bash
npm run build
npm run preview
```

## Data

`src/studies.mock.json` — **50 synthetic (mock) studies**. This is not real
research; it was generated by `scripts/generate-mock-studies.mjs`. Each study
has `id`, `title`, `authors`, `year`, `abstract`, and `status`. Regenerate with
`npm run generate:data`.

## Styling

Styled with **Tailwind CSS** (utilities in the JSX; config in `tailwind.config.js`,
PostCSS in `postcss.config.js`, directives in `src/index.css`). One deliberate
detail: the per-decision colours (include/exclude/flag) are stored as **complete,
static class strings** in the `DECISIONS` map. Tailwind scans source text for
literal class names, so a constructed string like `bg-${color}-700` would be
purged from the build — writing them out in full keeps them in the output.

## How it works

- **`useStudyDecisions.js`** — a custom hook that owns the reviewer's decisions
  and syncs them to `localStorage` on every change. It stores only the decisions
  map (`{ studyId: status }`), not the study text, so the source JSON stays the
  single source of truth. Reads are wrapped in `try/catch`; if storage is
  disabled or full the app keeps working in memory and warns the reviewer
  instead of crashing.
- **`StudyScreener.jsx`** — a two-pane layout: a navigable study list on the
  left and a reading pane on the right, with search, decision filters, and a
  progress bar.

## Keyboard accessibility (a graded requirement)

Everything works without a mouse:

| Key            | Action                                  |
| -------------- | --------------------------------------- |
| `↑` / `↓`      | Move between studies                    |
| `Home` / `End` | Jump to first / last                    |
| `I`            | Include                                 |
| `E`            | Exclude                                 |
| `F`            | Flag for discussion                     |
| `U`            | Clear (back to undecided)               |

Implementation details:

- The study list is an ARIA **`listbox`** with `role="option"` items and a
  **roving `tabindex`** (only the selected row is tabbable; arrows move focus).
- Decision shortcuts are handled by a window-level listener that is suppressed
  while the search box is focused, so typing a query never triggers a decision.
- Focus is never removed — `:focus-visible` shows a clear teal outline.
- An `aria-live="polite"` region announces each decision to screen readers
  (e.g. "Include: <title>. Study 4 of 50.").
- The layout collapses to a single column below 820px and respects
  `prefers-reduced-motion`.

## What I'd improve with more time

I'd add an undo (an accidental `E` is currently one keystroke with no take-back),
persist the last-selected study and active filter so a returning reviewer lands
where they left off, and virtualise the list so it stays smooth at thousands of
studies rather than fifty.
